git撤销更新

撤销暂缓区更新

使用”git add”把更新提交到了暂存区。这时”git status”的输出中提示我们可以通过git reset HEAD <file>...把暂存区的更新移出到WorkSpace中

撤销本地仓库更新

撤销提交有两种方式：使用HEAD指针和使用commit id

  1. HEAD指针
    在git中,HEAD指针指向当前分支中最新的提交。

  2. 使用commit id
    commit id可以使用git log命令查看

    –hard : 撤销并删除相应的更新(删除本地仓库,工作空间的更新)

    –soft : 撤销相应的更新,把这些更新的内容放到暂缓区(stage/index)中

恢复被撤销的提交可用git reflog查看仓库中所有分支的更新记录,包括已经撤销的更新

撤销(git add操作)

执行git rm --cached <file>命令

直接从暂存区删除文件，工作区则不做出改变
