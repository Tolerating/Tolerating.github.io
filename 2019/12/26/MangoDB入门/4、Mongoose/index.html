MongoDB入门 -- MongoDB基本操作

发表于 2019-12-26 | 分类于 后端学习 ， MongoDB入门 | 本文总阅读量 次
字数统计: 1.4k | 阅读时长 ≈ 5

Mongoose是一个让我们可以通过Node来操作MongoDB的模块。
Mongoose是一个对象文档模型（ODM）库，它对Node原生的MongoDB模块进行了进一步的优化封装，并提供了更多的功能。
在大多数情况下，它被用来把结构化的模式应用到一个MongoDB集合，并提供了验证和类型转换等好处。

安装Mongoose

官方文档

Mongoose并非Nodejs自带的模块,需要我们自行下载。

Mongoose的好处:

  • 可以为文档创建一个模式结构（Schema）
  • 可以对模型中的对象/文档进行验证。
  • 数据可以通过类型转换转换为对象模型
  • 可以使用中间件来应用业务逻辑挂钩
  • 比Node原生的MongoDB驱动更容易

Mongoose中为我们提供了几个新的对象:

  • Schema(模式对象)
    该对象定义约束了数据库中的文档结构
  • Model
    Model对象作为集合中的所有文档的表示，相当于MongoDB数据库中的集合Collection
  • Document
    Document表示集合中的具体文档，相当于集合中的一个具体的文档。

Schema

Mongoose 的一切始于 Schema。每个 schema 都会映射到一个 MongoDB collection ，并定义这个collection里的文档的构成。
简单来说就是创建一个约束对象,约束文档中属性的类型

name:String表示限定name属性的值只能为文本类型。
gender:{type:String,default:'男'}表示gender属性的类型为文本类型,默认值为男

Model

Models 是从 Schema 编译来的构造函数。 它们的实例就代表着可以从数据库保存和读取的documents。
Model代表的是数据库中的集合，从数据库创建和读取 document 的所有操作都是通过 model 进行的。

Model的方法

有了Model,我们就可以对数据库进行增删改查的操作了
Model所有方法

常用的几个方法:
Model.create(doc(s),function(err,candies){})
doc(s): 该参数表示可以传一个或多个文档(多个为数组)
返回值: Promise对象
例子:

Model.findById(id,[projection],[options],[callback])
根据文档的_id属性查询文档,返回的是一个对象

Model.findOne([conditions],[projection],[options],[callback])
查询符合条件的第一个文档,返回的是一个对象

Model.find([conditions],[projection],[options],callback])
查询符合条件的所有文档,返回的是一个数组

conditions: 查询的条件
projection: 投影,可以是一个字符串或对象。
例如{name:1,_id:0},意思是只返回name属性的值
‘name age’,意思是只返回name和age属性的值,但是还会额外返回一个_id属性的值,如不想返回_id属性的值,改为:name age -_id

options: 查询选项(skip和limit),例如:{skip:3,limit:4}

  • skip:跳过前几个数据
  • limit:限制返回文档的数量

callback: 回调函数,查询结果会通过回调函数返回,回调函数必须传

  • err参数:错误信息
  • docs参数: 返回的文档对象

Model.update(conditions,doc,[options],[callback]) 已经弃用,推荐使用下面两个
Model.updateMany(conditions,doc,[options],[callback])
Model.updateOne(conditions,doc,[options],[callback])*
用来修改一个或多个文档
conditions: 查询条件
doc: 修改后的对象
options: 配置参数
callback:回调函数

Model.remove(conditions,[callback]):删除查询到的所有文档
Model.deleteOne(conditions,[callback])
Model.deleteMany(conditions,[callback])
