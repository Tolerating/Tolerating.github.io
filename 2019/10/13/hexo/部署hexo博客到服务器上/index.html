部署hexo博客到服务器上

在将博客部署到coding和github上后,发现手机上访问会有很长一段时间不显示文章（一直刷新几次会正常显示）,后来在电脑上发现从远程仓库返回的utils.js不对导致了这个错误，网上百度并没有发现该错误的解决办法，最后实在没有办法，只能将hexo博客部署到服务器上（coding和github上的依旧保持更新）。

部署方法

网上部署hexo到服务器上的大致思路如下：

  1. 服务器端配置git私人仓库
  2. 使用nginx，将访问地址指向私人仓库中的博客地址
  3. 本地写好博客后，将生成好的博客上传至私人博客

我的部署方法是采用Jenkins远程拉取github上的博客，然后部署到nginx服务器上，其实跟上面讲到的方法大同小异（主要原因是因为我的服务器是windows的，ssh连接起来比较麻烦）

使用Jenkins部署博客到服务器上

下载安装Jenkins

Jenkins官网

我下载的是war包版(需要配置java环境),如果没有配置java环境可以下载windows版。

下面主要讲war包版的

war包版的运行起来十分简单:

在war包所在的目录运行命令窗口

命令行会有密码,复制改密码然后安装推荐插件

可能会有几个安装失败,提醒是否重试,可以点击一下重试,如果还是安装失败就直接忽略继续下一步

创建完用户后登陆Jenkins

配置jenkins

进入下列目录中配置Jenkins Location选项

注意: jenkins URL地址为服务器的内网地址，端口号和启动时的端口号保持一致

修改端口号：java -jar jenkins.war --ajp13Port=-1 --httpPort=6900

配置好后点击保存。

点击Manage Jenkins —-> Global Tool Configuration

查看Git选项是否配置了 安装推荐配置的时候默认是安装Git的,如果安装成功这里是会有配置的

这里我是用了自己安装的Git

点击Manage Jenkins —-> Configure Global Security

授权策略中把匿名用户具有可读权限勾上，把跨站请求伪造保护的勾去掉

不然github的webhook会报403

创建新任务

1. 点击 新建Item

2. 输入任务名,选择Freestyle project

添加描述

描述里输入对该任务的说明补充

配置源码管理

Repository URL是你放个人博客的仓库地址(注意不是源码!!!)

Credentials是连接仓库的凭证,没有的话点击添加:

输入登录仓库的用户名和密码

ID不用填

配置构建触发器

构建触发器勾选 触发远程构建

这里使用到仓库的webHook,即当仓库更新时会通过一个url触发构建的操作

URL的格式: Jenkins_URL/job/test/build?token=TOKEN_NAME

身份验证令牌可自定义填写

配置构建

window服务器选择第一个,Linux服务器选择第二个

上面的dos命令先是删除lkblog文件夹,然后复制Jenkins工作空间下的lkblog复制到nginx中,然后重启nginx服务器

最后点击保存

配置github的webhook

进入个博客的仓库,点击settings,点击WebHook,点击 Add webhook

Payload URL的值为: Jenkins_URL/job/test/build?token=TOKEN_NAME

注意 Jenkins_URL必须是外网地址

配置好Payload URL后点击 Add webhook

以上就打工搞成啦 (^_^)
