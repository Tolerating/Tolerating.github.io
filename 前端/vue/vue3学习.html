Composition API

Composition API为vue应用提供更好的逻辑复用和代码组织。

使用感受：将vue2中的option API(data,methods,computed等options)通过import的方式引入，这就是vue3的Composition API

Composition API可以和option API同时使用(Composition API会注入到option API中)

1.响应式数据reactive和ref

vue2中的响应式数据是通过defineProperty来实现的。
vue3中响应式数据是通过ES6的proxy来实现的。

在vue2中，我们创建响应式的数据是通过data函数return一个对象。

在vue3中引入一个reactive函数或ref函数来创建响应式数据。

  • reactive监听复杂类型的变化，其参数必须是一个对象(json/array)
  • ref监听某个变量的变化(可以是一个对象、数字、字符等等)

ref注意点：

  • 在vue（==template标签内==）中使用ref的值不用通过.value获取,例如在双括号中使用
  • 在js中使用ref的值必须通过.value获取

setup函数是组合API的入口函数

ref引用DOM元素的用法

1.1 递归监听

默认情况下，通过ref和reactive创建的响应式数据都是递归监听的（如果数据存在多层嵌套，reactive会将每层都包装成一个proxy对象），如果数据量比较大，就会非常消耗性能。可以使用shallowRef与shallowReactive

1.2 shallowRef与shallowReactive

shallowReactive只会把顶层的对象包装成proxy对象

shallowRef只会监听.value的变化（==这是因为Ref的本质就是一个reactive，ref(xx) -> reactive(value:xx)==）

应用场景：

  1. 一般情况下只使用ref和reactive即可。
  2. 只有在需要监听的数据量比较大的时候，才使用shallowRef或shallowReactive

shallowReactive创建的数据只会对顶层数据的修改做出响应

如果我们修改了shallowRef创建的数据(深层数据),想要界面做出响应，需要使用triggerRef函数

1.3 triggerRef

使用方法：在修改了shallowRef的深层数据后，调用triggerRef函数，传入被修改的shallowRef对象;

注意：Vue3只提供了triggerRef方法，没有提供triggerReactive方法，所以如果是reactive类型的数据，那么是无法主动触发界面更新的。

1.4 toRefs()

该方法用来将数据转换成一个ref对象

vue3中的方法

在vue3中如果想定义了点击事件的方法，我们可以直接在setup函数中定义方法，然后return方法名即可。

setup函数

setup执行时机

setup函数在beforeCreate周期函数和Created周期函数之间执行。

beforeCreate:表示组件刚刚被创建出来，组件的data和methods还没有初始化好。

Created:表示组件刚刚被创建出来，并且组件的data和methods已经初始化好。

setup注意点

  • 由于在执行setup函数的时候，还没有执行Created生命周期方法，所以在setup函数中，是无法使用data和methods
  • 由于setup函数中不能使用data和methods，所以setup函数中不能使用this(打印this的结果为undefined)
  • setup函数只能是同步的。

setup函数内部可调用的生命周期挂钩

  • onBeforeMount
  • onMounted
  • onBeforeUpdate
  • onUpdated
  • onBeforeUnmount
  • onUnmounted
  • onErrorCaptured
  • onRenderTracked
  • onRenderTriggered

这些函数在调用的时候接受一个回调，该回调将在组件调用该挂钩的时候执行。
