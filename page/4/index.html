centos7相关命令

centos后台运行jar包

  1. java -jar *.jar

    特点：当前ssh窗口被锁定，可按CTRL + C打断程序运行，或直接关闭窗口，程序退出

  2. java -jar *.jar &

    &代表在后台运行。
    特定：当前ssh窗口不被锁定，但是当窗口关闭时，程序中止运行。

  3. nohup java -jar *.jar &

    nohup 意思是不挂断运行命令,当账户退出或终端关闭时,程序仍然运行

结束后台任务

通过ps aux命令查看进程号PID，然后执行 kill -9 PID

.not

否定链中随后的所有断言

.deep

全等，但不同于(==),主要用于对象的比较

.nested

嵌套.nested (不能和.own一起用)———–在property和include之前

.own

忽略继承的属性(own可理解为自己的)———–在property和include之前

不可以.nested一起用

.ordered

导致链中跟随的所有.members断言要求成员具有相同的顺序。

.any

导致链中跟随的所有.keys断言只要求目标具有至少一个给定的键。这与.all相反，.all要求目标具有所有给定的键。

有关何时使用.any或.all的指导，请查看.keys

.all

导致链中跟随的所有.keys断言要求目标具有所有给定的键。这与.any正好相反，后者只要求目标至少具有一个给定的键。

注意，当在链的前面添加.all和.any时，默认情况下使用.all。但是，通常最好还是添加.all，因为它提高了可读性。

.a(type[,msg])

类型检查: 断言目标的类型等于给定的字符串类型。类型不区分大小写。

type如果是以元音开头的,用.an()。 其他的用.a()

.include(val[,msg])

断言目标中是否包含给定的值(默认情况下使用严格相等,但是可以使用.deep)

.ok

断言目标是一个真实值(在布尔上下文中被认为是真实的)。然而，通常最好断言目标是严格地(===)或完全等于其期望值。

.true

断言目标严格地(===)等于true。

.false

断言目标严格地(===)等于false

.null

断言目标严格地(===)等于null

.undefined

断言目标严格地(===)等于undefined

.NaN

断言目标严格地(===)等于NaN

.exist

断言目标不是严格地(===)等于null或undefined。然而，通常最好是断言目标等于它的期望值。

.empty

当目标是一个字符串或数组时，.empty断言目标的length属性严格地(===)等于0。

.arguments

断言目标是arguments对象。

.equal(val[,msg])

断言目标严格地(===)等于给定的val

.eql(obj[,msg])

断言目标与给定的obj深度相等。
